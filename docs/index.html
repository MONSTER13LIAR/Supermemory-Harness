Install

Install once. Run smctl from then on.

Harness is distributed through npm. After the global install, users do not need to copy a GitHub URL again.

First install

# Install the CLI
npm install -g supermemory-harness

# Activate the Harness workflow
smctl install

Use this after Supermemory Local is installed on the same machine.

Normal workflow

smctl supermemory start
smctl ui
smctl verify --timeout-ms 60000
smctl trust --probe
smctl gate

For one-off usage without a global install: npx -y supermemory-harness install.

Architecture

A support layer, not a replacement.

Supermemory Local remains the private memory engine at localhost:6767. Harness adds the operational layer that makes that engine easier to recommend to users and agents.

Agent memory path

Agents should not blindly trust memory. Harness gives them checks before edits, tests, migrations, compaction, or handoff.

Why this exists

Local memory becomes much more useful when users can see whether it is healthy, scoped, searchable, and safe to rely on.

  • Verify: proves write and recall work right now.
  • Trust: surfaces failed writes, drift, duplicates, vague memories, and processing issues.
  • Repair: gives ordered recovery steps without unsafe replay.
  • Guard: reviews risky writes before they become durable memory.
  • Genome: personalizes future memory behavior from the user's stored memory mix.
