Problem

Local memory is valuable only when it can be trusted.

Supermemory Local handles the hard memory engine work on the user's machine. Harness focuses on the operational questions that appear immediately after install.

  • Is the local server reachable?Harness checks the dashboard, OpenAPI routes, MCP surface, processing path, and project setup.
  • Is recall actually working?smctl verify writes a harmless marker, waits for processing, searches it back, and checks scoping.
  • Can an agent rely on this memory?Session and gate commands return pass, warn, or block decisions before important edits or handoffs.
  • What should the user do next?Repair, trust, and launch commands turn raw state into ordered next actions instead of raw logs.
Architecture

A companion layer, not a replacement.

Harness talks to the real Supermemory Local server and keeps the memory engine local. It adds diagnostics, policy, and agent lifecycle checks around that engine.

Proof path

What a user sees after installation.

The walkthrough is intentionally operational: install, open the dashboard, prove recall, inspect trust, repair when needed, and connect agents.

First-run clarity

The install path checks Local state, project scope, dashboard readiness, agent bridge setup, and the next command.

Dashboard command center

smctl ui keeps the Supermemory dashboard visible while adding Harness status and next-action controls.

Recall verification

smctl verify proves write, processing, search, project scoping, recall canaries, and language recall.

Trust before reliance

smctl trust --probe gives a live memory score and surfaces failed writes, drift, duplicates, and recall issues.

Safe repair plan

The repair wizard separates runtime failures from replayable memory problems and avoids unsafe changes.

Memory Genome

Harness classifies the user's memory mix and creates a local policy for cleaner future memory behavior.

Install

Install once. Run smctl from then on.

Use Harness on the same machine as Supermemory Local. The CLI remains local and communicates with the local server by default.

# Install globally
npm install -g supermemory-harness

# Activate the Harness workflow
smctl install

After global install, users do not need to copy the GitHub repository URL again.

# Normal local workflow
smctl supermemory start
smctl ui
smctl verify --timeout-ms 60000
smctl trust --probe
smctl gate

For one-off usage without global install, run npx -y supermemory-harness install.
